# index-wap-archive

## 项目简介

`index-wap-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的集合存储与展示空间。

适合存放各类前端页面、移动端适配页面、活动页、信息页等，方便通过 GitHub Pages 或其他静态托管服务进行访问与分享。

## 目录说明

```
index-wap-archive/
├── pages/          # 存放各个独立 HTML 页面及其相关资源
├── assets/         # 公共资源文件（如样式、脚本、图片等）
├── index.html      # 入口页面（可选，用于展示页面列表）
└── README.md       # 本文件
```

- `pages/`：每个子文件夹或文件代表一个独立的页面归档。
- `assets/`：存放多个页面可能共用的 CSS、JavaScript 或图片资源。
- `index.html`：可选的首页，用于生成页面索引或导航。

## 页面归档说明

本仓库中的页面均为独立的 HTML 文件，彼此之间无依赖关系。每个页面可独立访问，适合用于：

- 临时活动页面存档
- 移动端适配页面展示
- 前端原型或测试页面
- 信息展示页

归档时建议在每个页面文件夹内附带必要的资源文件，以保证页面在离线或迁移环境下仍可正常显示。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新文件夹，放入 `index.html` 及相关资源。
- 删除页面：直接移除对应文件夹即可。
- 修改页面：更新对应文件夹内的文件，保持目录结构清晰。
- 公共资源：如有多个页面共用资源，可统一放在 `assets/` 下，页面内通过相对路径引用。

本仓库主要面向个人或小团队使用，不保证长期在线或持续更新。建议使用者自行备份重要页面。
